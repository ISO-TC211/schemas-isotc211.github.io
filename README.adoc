== Introduction

The ISO/TC 211 XML Schemas site publishes XML schemas from ISO/TC 211 standards at
https://schemas.isotc211.org. Each schema package has:

* **Downloadable XSD files** at `https://schemas.isotc211.org/schemas/{standard}/{part}/{prefix}/{version}/`
* **Interactive schema browsers** (lutaml-xsd SPA) per standard+part+version
* **A card-based index page** with ISO visual styling

This repository is jointly managed by ISO/TC 211 and operated by Ribose under
the ISO/TC 211 Harmonized Resources Maintenance Agency (HRMA) run by the Swedish
Institute of Standards (SIS).


== Building the site

[source,sh]
----
# Full build (XSD downloads + index page + schema browsers)
make all

# Generate per-package config files from the manifest
make configs

# Build a single LXR package
make build/19136-gml-1.0.lxr

# Generate a single schema browser
make site/19136-gml-1.0.html

# Build just the index page
make site/index.html

# Local dev server on http://localhost:4000
make serve

# Clean all build artifacts
make clean
----

=== Prerequisites

* Ruby 3.3 with Bundler
* Node.js 20 with npm
* Git submodule (`schemas/`) initialized: `make update-init`


== Project Structure

* `schemas/` — Git submodule (ISO-TC211/schemas), contains XSD files + `lxr_packages.yml`
* `vendor_schemas/` — Helper schemas (xlink, w3c, OGC sensorML/sweCommon)
* `lxr_packages.yml` — Local copy of the package manifest (fallback until submodule is updated)
* `generate_configs.rb` — Generates lutaml-xsd configs + `schemas_index.json` from the manifest
* `configs/` — Auto-generated lutaml-xsd configs (generated)
* `build/` — Intermediate LXR packages (generated)
* `site/` — Final deployable output (generated)
** `index.html` — Card-based index page (Vite + Tailwind CSS)
** `schemas/` — XSD files for download
** `*.html` — Per-standard schema browsers
** `schemas_index.json` — Package data for the frontend
* `frontend/` — Vite + Tailwind CSS v4 index page source
* `Makefile` — Build orchestration
* `.github/` — GitHub Actions workflows


== Schema Documentation (lutaml-xsd)

Schema documentation is generated using the `lutaml-xsd` gem, which creates
interactive Vue.js SPA browsers from XSD files.

One LXR package is created per standard+part+version (~30 packages).
Packages defined as broken are temporarily skipped for SPA generation
(see `SPA_SKIP` in the Makefile) but XSD files remain downloadable.

[source,sh]
----
# Generate config files from the manifest
ruby generate_configs.rb

# Build a single schema package
make build/19136-gml-1.0.lxr

# Generate a single schema browser
make site/19136-gml-1.0.html
----


== Package Manifest

The canonical package manifest (`lxr_packages.yml`) lives in the
https://github.com/ISO-TC211/schemas[schemas submodule repo].
File paths are relative to that repo's root (e.g., `19103/-/gco/1.2.0/gco.xsd`).

Schema authors validate the manifest with:

[source,sh]
----
cd schemas/
bundle exec hrma lxr validate
----

This ensures all referenced XSD files exist before the site builds.
The site repo reads the manifest from the submodule and adds
schema location mappings, appearance config, and build settings.
